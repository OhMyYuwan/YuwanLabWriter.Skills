# Rebuttal Playbook

## Strategy Before Writing

Do not start by drafting polite sentences. First understand what each reviewer is trying to decide.

Build:

```text
ID | Reviewer | Comment summary | Category | Severity | Stance | Evidence needed | Manuscript change | Space budget
```

## Reviewer Priorities

- Positive reviewer: protect support; answer cleanly without overcomplicating.
- Borderline reviewer: spend the most care; remove uncertainty and show concrete changes.
- Negative reviewer: respond with evidence, correct misunderstandings, and make the AC able to trust your answer.

## Six Objection Types

### Novelty

Respond by restating the insight, not only the component list. If accused of simple A+B, explain why A+B is nontrivial and what fails without the proposed design.

### Factual Error / Misunderstanding

- If the paper is wrong: acknowledge, fix, cite the change.
- If the reviewer misunderstood: clarify and revise the manuscript so the misunderstanding is less likely.
- If the reviewer is factually wrong: politely correct with evidence and consider AC/AE message.

### Marginal Improvement

Use statistical stability, multiple datasets, fair-comparison controls, compute/parameter trade-off, or practical significance.

### Missing Experiment

If feasible, add it. If not feasible or not allowed by rebuttal policy, explain why and state what will be added in the final/revision version if appropriate.

### Presentation

Fix quickly and cite locations. This is low-cost trust-building.

### Scope

Cite venue/journal fit using recent comparable publications when possible. If the scope is truly weak, advise resubmission to a better venue.

## Response Formula

```text
Comment:
> [reviewer comment]

Response:
Thank you for [specific point]. [Agree / clarify / respectfully disagree]. [Evidence, data, reasoning, or limitation]. We have revised [section/line/table/figure] to [specific change].
```

## AC/AE Message Conditions

Use sparingly for:

- Clear factual error.
- Internal contradiction.
- Review confidence mismatch.
- Possible conflict or bias.
- Reviewer demands outside venue policy or scope.

Tone: factual, concise, non-accusatory.
